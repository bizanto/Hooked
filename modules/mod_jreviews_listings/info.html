LISTINGS MODULE PARAMETERS HELP
Everywhere Extension
This is a universal module that works not only with core jReviews listings, but also with 3rd party components integrated via the Everywhere addon.
This parameter should not be left empty. You must specify a component (i.e. com_content, com_virtuemart, com_sobi2, etc.).
jReviews Template Suffix
This parameter allows you to use a different theme file for each published instance of the module. For example you can use a suffix of "_latest" and a suffix of "_toprated" and then copy and rename the /modules/listings.thtml theme file to listings_latest.thtml and listings_toprated.thtml.
Category Auto-detect
This parameter will work only if an extension is filled-in for Everywhere Extension. When looking at a category page for the chosen extension, jReviews will try to determine the category id and show only results for that category. If enabled, this setting overrides all the ID settings below it.
Directory, Section, Category, Listing IDs
Filter the results by specifying the IDs. For each parameter you can enter more than one ID separated by comma, NO spaces.
(com_content) = only works if Everywhere Extension = com_content
(everywhere) = works for any component used in Everywhere Extension
